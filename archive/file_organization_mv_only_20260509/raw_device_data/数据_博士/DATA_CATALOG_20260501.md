## Format Issue

`.qivd` is a **proprietary binary format** (Keysight B291x "Utility Software Save File Header"). Direct binary parsing was attempted but the data encoding (compression, chunking, or custom serialization) is not immediately reverse-engineerable.

**Options for Codex / next step:**

1. **Preferred:** Export to CSV from Keysight B2900A/B2910A control software, then replace `.qivd` files.
2. **Alternative:** Install `Keysight IO Libraries Suite` and use its COM/SCPI interface to re-read saved files.
3. **Fallback:** If the original acquisition PC has the Keysight software, use "Save As CSV" on all files and re-upload.

**Without format conversion, numerical analysis is blocked.**
