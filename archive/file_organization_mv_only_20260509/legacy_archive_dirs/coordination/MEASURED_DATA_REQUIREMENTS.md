## 6个关键参数

### 1. 电导下限 ($G_{min}$) 和 电导上限 ($G_{max}$)

| 项目 | 说明 |
|:-----|:-----|
| **物理意义** | 器件可达到的最低和最高电导状态，决定权重映射的动态范围 |
| **系统影响** | 影响量化精度和信噪比；论文中Zhang 2025 profile使用G_max/G_min=47.3 |
| **测试方法** | 脉冲编程扫描：施加递增/递减脉冲，记录稳定电导值 |
| **输出格式** | G_min [S], G_max [S], 或比值 G_max/G_min |
| **样本量** | 至少3个器件，每个扫描3个周期 |

---

### 2. 稳定状态数 ($n_{states}$)

| 项目 | 说明 |
|:-----|:-----|
| **物理意义** | 在G_min到G_max范围内可区分的稳定电导状态数量 |
| **系统影响** | 决定有效比特精度；论文使用4-bit (16状态)，ADC是实际瓶颈 |
| **测试方法** | 多电平编程：将器件编程到目标电导，测量保持稳定状态的成功率 |
| **判别标准** | 状态电导分布CV < 5%，相邻状态无重叠 |
| **输出格式** | 整数，如16、32、64等 |

---

### 3. 器件间散布 ($\sigma_{D2D}$)

| 项目 | 说明 |
|:-----|:-----|
| **物理意义** | 同一批次不同器件在编程到同一状态时的电导差异，反映工艺均匀性 |
| **系统影响** | **极高影响** — 这是Ensemble HAT要解决的核心问题；论文中10% D2D导致标准HAT崩溃至10%，Ensemble HAT恢复至86.37% |
| **测试方法** | 阵列统计：编程M个器件到同一状态，统计电导分布 |
| **计算公式** | $\sigma_{D2D} = \text{std}(G_1, ..., G_M) / \text{mean}(G)$ |
| **样本量** | 至少20个器件，覆盖阵列不同位置 |
| **输出格式** | 相对标准差，如0.10表示10% |

---

### 4. 读写循环波动 ($\sigma_{C2C}$)

| 项目 | 说明 |
|:-----|:-----|
| **物理意义** | 同一器件多次编程到同一目标状态时的电导波动 |
| **系统影响** | **低影响** — EXP-A显示在D2D主导时C2C几乎不影响系统级准确率；可用文献代理 |
| **测试方法** | 重复编程测试：重复编程N次，每次读取电导 |
| **计算公式** | $\sigma_{C2C} = \text{std}(G_1, ..., G_N) / \text{mean}(G)$ |
| **输出格式** | 相对标准差，如0.05表示5% |
| **优先级** | P1（有则更好，无则可用文献值5%） |

---

### 5. 时间保持力 ($\tau$)

| 项目 | 说明 |
|:-----|:-----|
| **物理意义** | 电导随时间衰减的时间常数；论文使用双指数模型简化 |
| **系统影响** | 决定推理有效期；论文显示1000s后保持约79%准确率 |
| **测试方法** | 长期保持测试：编程后连续监测电导变化 |
| **简化模型** | 单指数近似即可：$G(t) = G_0 \cdot e^{-t/\tau}$ |
| **关键时间点** | 1s, 10s, 100s, 1000s |
| **输出格式** | 时间常数 $\tau$ [秒]，如0.5s |

**注**: 如需更精确，可提供双指数参数 ($\tau_1, \tau_2, A_0$)，但单指数足以支撑论文结论。

---

### 6. 写入非线性 ($NL_{LTP}$ 和 $NL_{LTD}$)

| 项目 | 说明 |
|:-----|:-----|
| **物理意义** | 连续施加脉冲时的电导更新曲线非线性度；NL=1为线性，NL>1为超线性 |
| **系统影响** | **极高影响** — 论文发现NL=2.0是训练硬边界(准确率27.72%)，决定HAT可行性 |
| **测试方法** | 增量电导测量：从不同初始G值开始，施加相同脉冲，测量ΔG vs G关系 |
| **计算公式** | $\Delta G \propto G^{NL}$，拟合log(ΔG) vs log(G)的斜率 |
| **区分方向** | 分别测试增强(LTP)和抑制(LTD)方向 |
| **输出格式** | NL_LTP [标量]，NL_LTD [标量，通常负值] |

## 优先级分级

| 优先级 | 参数 | 影响 | 可代理？ |
|:------:|:-----|:-----|:--------:|
| **P0 (必需)** | $\sigma_{D2D}$ | 极高 | ❌ 必须实测 |
| **P0 (必需)** | $NL_{LTP}, NL_{LTD}$ | 极高 | ❌ 必须实测 |
| **P0 (必需)** | $G_{min}, G_{max}$ | 高 | ⚠️ 最好用实测 |
| **P1 (推荐)** | $\tau$ | 中 | ✅ 可用文献代理 |
| **P1 (推荐)** | $n_{states}$ | 低 | ✅ 可用4-bit假设 |
| **P2 (可选)** | $\sigma_{C2C}$ | 极低 | ✅ 可用5%代理 |

## 最小可行数据集

如果只能测量部分参数，**最简配置**为：

```json
{
  "must_have": {
    "sigma_d2d": "器件间失配，驱动Ensemble HAT必要性",
    "nl_ltp/ltd": "写入非线性，决定训练可行性边界"
  },
  "should_have": {
    "g_range": "电导动态范围，影响绝对准确率数值"
  },
  "can_proxy": {
    "tau": "保持时间，可用文献值0.5s",
    "n_states": "状态数，可用4-bit(16)假设",
    "sigma_c2c": "循环波动，可用5%代理"
  }
}
```

**结论**: 即使只测得$\sigma_{D2D}$和$NL$，论文核心结论（Ensemble HAT有效性、NL=2.0硬边界）仍有实验支撑。
